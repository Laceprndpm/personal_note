struct LazySegmentTree {
    int          n;
    vector<Info> info;
    vector<i64>  leftB;
    vector<i64>  rightB;

    LazySegmentTree() : n(0) {}

    LazySegmentTree(int _n, Info _v = Info())
    {
        init(_n, _v);
    }

    template <class T>
    LazySegmentTree(vector<T> _init)
    {
        init(_init);
    }

    void init(int _n, Info _v = Info())
    {
        init(vector(_n, _v));
    }

    template <class T>
    void init(vector<T> _init)
    {
        n = _init.size() - 1;
        info.assign(4 << __lg(n), Info());
        leftB.resize(4 << __lg(n));
        rightB.resize(4 << __lg(n));
        function<void(int, int, int)> build = [&](int p, int l, int r) {
            leftB[p]  = l;
            rightB[p] = r;
            if (r == l) {
                info[p] = _init[l];
                return;
            }
            int m = (r - l) / 2 + l;
            build(2 * p, l, m);
            build(2 * p + 1, m + 1, r);
            pull(p);
        };
        build(1, 1, n);
    }

    void pull(int p)
    {
        info[p] = info[2 * p] + info[2 * p + 1];
    }

    Info rangeQuery(int p, int l, int r, int x, int y)
    {
        if (l >= x && r <= y) {
            return info[p];
        }
        if (l > y || r < x) {
            return Info();
        }
        int m = (r - l) / 2 + l;
        return rangeQuery(2 * p, l, m, x, y) + rangeQuery(2 * p + 1, m + 1, r, x, y);
    }

    Info rangeQuery(int l, int r)
    {
        if (l > r) {
            return Info();
        }
        return rangeQuery(1, 1, n, l, r);
    }
};

## editorial
考虑题目的朴素做法：直接把列切出来，然后数这个子矩阵的连通块数。常规的两种方法是 DFS/BFS 或 DSU。我个人觉得 DSU 更容易扩展到完整问题。

用 DSU 统计连通块的方法如下：初始化时没有边，每个空单元格单独作为一个连通块；然后依次加入边。每条边连接两个相邻单元格（上下或左右）。如果边连接了两个不同的连通块，就把它们合并，同时连通块数减一。

因此，某一列区间上的连通块数 = 该区间中空单元格个数 − 区间中 **会真正合并的边数**（即这个区间自己的生成森林里的边数）。

现在试图把这个算法扩展到整个问题。理想情况是：把整张矩阵的生成森林先建好，然后对每个查询直接输出：

> 空单元格数 − 查询区间里的生成森林边数

可惜没这么简单。对于完全位于查询区间内部的连通块，这没问题；但是如果一个连通块跨越查询边界，它可能在区间内部仍然连着，也可能被切断。如果能判断它是否断裂，就能修正答案。

可以有很多调整方式，这里给出我认为代码最简洁的一种。向 DSU 加边时，我们按以下顺序处理整张图：

1. **从左到右列序**
    
2. 在每一列内，先加所有纵向边（顺序随意）
    
3. 然后加从该列指向左边那一列的所有横向边（顺序随意）
    

如果从第 1 列开始做这个过程，那么对于所有 **l = 1** 的查询都能直接回答。因为随着列不断加入，DSU 构建的生成森林在任何时候都是当前前缀列的正确生成森林，所以某段的答案确实是“空格数量 − 边数”。

现在分析：从第 1 列开始，与从任意列 l 开始的区别在哪里。

观察列 l：

- 如果它包含 1 个或 3 个空格，或包含 2 个相邻的空格，则无论左边发生了什么，这些空格都必然属于同一连通块；
    
- 如果没有空格，左边也不影响任何东西。
    

因此，如果列 l 不是“101”形状，那么以第 1 列为起点构造的生成森林，在从这个 l 开始的任何查询中都是正确的。

唯一需要额外处理的是：**列 l 只有第 1 行和第 3 行是空格（“101”列）**。这两个空格会不会提前在左边就被连通？我们无法知道。

### **分类讨论**

设查询最左列为 L：

- **如果 L 本身不是“101”，无须额外加任何成分。**
    
- **如果整个查询都是“101”列** → 答案是 2（行 1 和行 3 各一个连通块）。
    
- 如果第一次出现的非“101”列是“111”列 → 不需加，因为前面的“101”会被它的连通块吸收。
    
- 如果该列是“000”或“010” → 需要额外加 2（行 1 和行 3 都不会被并入任何东西）。
    
- 否则 → 加 1。
    
空格数和边数的区间和可以用前缀和预处理。最近的非“101”列也能线性求出。

总复杂度：**O(n·α(n) + q)**。
